คู่มือการใช้งาน Terumo BR-500 Simulator & Reader

เอกสารนี้อธิบายถึงขั้นตอนการตั้งค่าและใช้งานโปรแกรมจำลองเครื่องวัดความดัน Terumo BR-500 รวมถึงแนวทางในการนำ Simulator ตัวนี้ไปใช้เป็นเครื่องจำลองสำหรับการพัฒนาแอปพลิเคชันบน Android

1. ภาพรวมของระบบ (System Architecture)

ระบบประกอบด้วย 3 ส่วนหลัก ได้แก่ ตัวส่ง (Simulator), ตัวเชื่อมประสาน (Virtual COM Port), และตัวรับ (Reader หรือ Android App)

2. การติดตั้งและตั้งค่าพอร์ตจำลอง (Virtual COM Port Setup)

ในการพัฒนาโปรแกรมที่ต้องเชื่อมต่อกับอุปกรณ์ฮาร์ดแวร์ผ่าน Serial Port นักพัฒนาจำเป็นต้องมี Virtual COM Port Bridge เพื่อจำลองสายเคเบิลรับส่งข้อมูลภายในเครื่องคอมพิวเตอร์ตัวเอง ทำให้สามารถใช้รันโปรแกรม Simulator (ผู้ส่ง) และโปรแกรมแอปพลิเคชัน (ผู้รับ) พร้อมกันได้โดยไม่ต้องมีฮาร์ดแวร์จริง

ใช้โปรแกรม HHD Virtual Serial Port Tools

จุดเด่น: มีหน้าตา UI ใช้งานง่าย เหมาะสำหรับการนำไปจำลองพอร์ตให้แอปพลิเคชัน (Android) หรืออ่านค่าด้วยโค้ด Python

ขั้นตอนการตั้งค่า (อ้างอิงตามภาพด้านบน):

  1. ดาวน์โหลดโปรแกรมจากเว็บไซต์ (ลิงก์ดาวน์โหลด) และทำการติดตั้ง
  2. เปิดโปรแกรม มองหากล่อง "Local Bridges" ที่อยู่มุมซ้ายบน
  3. กดปุ่มเครื่องหมายบวกสีเขียว (+) Create New Bridge ที่อยู่ด้านล่างของกล่องนั้น
  4. เลือกระบุพอร์ตซ้ายเป็น COM3 และระบุพอร์ตขวาเป็น COM4 จากนั้นกดปุ่ม Apply
  5. เมื่อเห็นรายการ COM3 ↔ COM4 ปรากฏขึ้นในกล่องสีขาว ถือว่าการตั้งค่าเสร็จสมบูรณ์ พอร์ตทั้งสองทะลุถึงกันแล้ว

3. วิธีการใช้งาน Simulator และ Reader

โปรแกรมทั้งสองทำงานแบบ Standalone (ไฟล์ .exe) สามารถเปิดใช้งานได้ทันทีโดยไม่ต้องติดตั้ง

ฝั่งเครื่องจำลอง (Simulator)

  1. เปิดไฟล์ terumo_simulator.exe
  2. เลือก Port: COM3 และตั้งค่า Baud: 19200
  3. กดปุ่ม Connect
  4. สามารถติ๊กเลือก "สุ่มค่าใหม่ทุกครั้งที่ส่ง" เพื่อให้ข้อมูลเปลี่ยนไปเรื่อยๆ โดยอัตโนมัติ
  5. ติ๊กที่ Auto-send every X sec เพื่อให้โปรแกรมยิงข้อมูลออกไปอย่างต่อเนื่อง

ฝั่งโปรแกรมอ่านค่า (Reader) - สำหรับใช้ทดสอบ

  1. เปิดไฟล์ terumo_reader.exe
  2. เลือก Port: COM4 และตั้งค่า Baud: 19200 (ต้องตรงกับ Simulator)
  3. กดปุ่ม Connect
  4. เมื่อ Simulator ส่งข้อมูลมา ตัว Reader จะถอดรหัสและดึงตัวเลขความดัน (SYS, DIA, PULSE) มาแสดงผลที่หน้าจอทันที

4. การนำไปใช้พัฒนาแอปพลิเคชัน Android

คุณสามารถใช้ terumo_simulator.exe เป็นเสมือนเครื่อง Terumo ของจริง เพื่อยิงข้อมูลความดันเข้าแอปพลิเคชัน Android ของคุณที่กำลังพัฒนาอยู่ได้

4.1 รูปแบบข้อมูลที่ส่งออก (Protocol Format)

ข้อมูลที่ Simulator ยิงออกมาจะเป็นรูปแบบ R1 Frame มาตรฐานของ Terumo:

  • รูปแบบ: <STX> R1,ID,Date,Time,SYS,MAP,DIA,PULSE,0000,0000,00000,000 <ETX> <BCC>
  • การถอดรหัสในแอป: เมื่อแอป Android รับข้อมูลมา ให้แยกข้อความด้วยเครื่องหมายจุลภาค (,)
    • Index 4 = SYS (Systolic)
    • Index 5 = MAP
    • Index 6 = DIA (Diastolic)
    • Index 7 = PULSE

4.2 การจำลองพอร์ตสำหรับ Android Emulator

หากคุณเขียนแอปบน Android Studio และใช้ Android Emulator ในการรันแอป คุณสามารถส่งผ่าน (Map) พอร์ต COM4 เข้าไปยัง Emulator ได้:

  1. ปิด Android Emulator
  2. รัน Emulator ผ่าน Command Line โดยเพิ่ม Flag -qemu -serial COM4
    ตัวอย่างคำสั่ง: 
    emulator -avd Pixel_4_API_30 -qemu -serial COM4
  3. ในโค้ดแอป Android ให้เปิดการเชื่อมต่อพอร์ตที่ /dev/ttyS1 หรือ /dev/ttyS2 (ซึ่งเป็นพอร์ตเสมือนที่ถูกโยงกับ COM4 ของ PC)

4.3 การเชื่อมต่อกับอุปกรณ์โทรศัพท์ Android ของจริง (Physical Device)

หากทดสอบบนมือถือ/แท็บเล็ตจริง คุณจะต้องใช้สายเคเบิลแปลงสัญญาณ (USB to RS232) ดังนี้:

  1. นำหัวแปลง USB to Serial (RS232) เสียบเข้ากับพอร์ต USB ของ PC (สมมติได้ชื่อพอร์ตเป็น COM5)
  2. นำสาย USB OTG เสียบเข้ากับมือถือ Android และต่อเข้ากับหัวแปลง USB to Serial อีกเส้นหนึ่ง
  3. นำปลายสาย Serial (หัว DB9) ของทั้งสองฝั่งมาต่อกัน (อาจต้องใช้หัวแปลง Null Modem ตัวเมีย-ตัวเมีย)
  4. เปิดโปรแกรม HHD Virtual Serial Port Tools แล้วสร้าง Bridge เชื่อมระหว่าง COM3 (ฝั่ง Simulator) เข้ากับ COM5 (ฝั่ง USB ที่ต่อออกไปมือถือ)
  5. เปิดแอปบน Android (อาจใช้ไลบรารีอย่าง usb-serial-for-android) สั่งให้อ่านข้อมูลจากสาย USB OTG ก็จะได้รับข้อมูลที่ส่งมาจาก Simulator ทันทีครับ

สรุป Workflow ในการทำงานพัฒนาแอป

  1. เปิด HHD จำลองพอร์ต
  2. เปิด Simulator ปล่อยรันสุ่มค่าทิ้งไว้
  3. รันโปรเจกต์บน Android Studio ดักจับข้อมูล Serial Port
  4. เขียนโค้ดในแอป Android ให้ตัดคำด้วยลูกน้ำ (,) และนำค่าความดันไปโชว์ใน UI ของแอป
